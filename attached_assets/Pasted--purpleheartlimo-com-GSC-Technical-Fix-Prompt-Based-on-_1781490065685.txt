## TASK 2 — FIX THE SPA ROUTER LEAK (HTML tag names as URLs)

### The problem:
GSC shows this URL being crawled:
https://purpleheartlimo.com/html/head/title

This is a known SPA (Single Page Application) router bug where
the router is treating HTML tag names as route paths. This was
identified in a previous audit as an existing issue.

### Step 1 — Find the router configuration:
Look for: App.tsx, App.jsx, router.tsx, routes.tsx, main.tsx

### Step 2 — Add a catch-all 404 handler:
The router needs to catch any route that doesn't match a known
page and return a proper 404 instead of rendering the page.

In React Router:
```javascript
// Add as the LAST route in your routes array
{
  path: '*',
  element: <NotFound /> // or redirect to homepage
}
```

In Next.js — create: pages/404.tsx or app/not-found.tsx

### Step 3 — Block HTML tag routes in robots.txt:
Open public/robots.txt and add:
```
Disallow: /html/
Disallow: /head/
Disallow: /body/
```

### Step 4 — Add 404 redirect for the leaked URL in _redirects:
```
/html/*   /   404
/head/*   /   404
/body/*   /   404
```

## TASK 3 — FIX THE WWW REDIRECT LEAK (36 pages affected)

### The problem:
GSC confirms both www versions still being crawled as of June 10:
- http://www.purpleheartlimo.com/
- https://www.purpleheartlimo.com/
- https://www.purpleheartlimo.com/blog/dallas-nightlife-guide-uptown-deep-ellum-by-limo
- Multiple www.purpleheartlimo.com/blog/* URLs

### Step 1 — Find current _redirects file:
Open _redirects (should be in /public or project root).

### Step 2 — Check if www redirect rule exists:
Look for a line like:
```
https://www.purpleheartlimo.com/*  https://purpleheartlimo.com/:splat  301!
```

### Step 3 — If missing or not using splat, fix it:
The rule MUST use :splat to pass the full path through.
Replace any incomplete www redirect with:

```
# www to non-www — catches ALL paths including blog posts
https://www.purpleheartlimo.com/*  https://purpleheartlimo.com/:splat  301!
http://www.purpleheartlimo.com/*   https://purpleheartlimo.com/:splat  301!
http://purpleheartlimo.com/*       https://purpleheartlimo.com/:splat  301!
```

The `!` at the end forces the redirect even if another rule
would match — critical for Netlify to override other rules.

### Step 4 — Verify the specific URLs that were crawled redirect:
After deploying, test these URLs return 301 → non-www:
- www.purpleheartlimo.com/blog/dallas-nightlife-guide-uptown-deep-ellum-by-limo
- www.purpleheartlimo.com/ (homepage)

- /blog/misconceptions-about-limousine-services--what-you-need-to-know
- /blog/how-to-create-a-lead-capture-system-for-limousine-services
- /blog/why-choose-purple-heart-limousine-for-your-next-austin-event-

These are legitimate content pages that should be indexable.

### Step 1 — Open robots.txt and check for any rules that
might accidentally block /blog/* paths.

### Step 2 — Common cause: overly broad Disallow rule:
If robots.txt has something like:
```
Disallow: /blog/misconceptions*
```
or any pattern that matches these slugs, remove or fix it.

### Step 3 — The only things that SHOULD be blocked in robots.txt:
```
User-agent: *
Disallow: /admin/
Disallow: /api/
Disallow: /cdn-cgi/
```

Blog posts should NEVER be in Disallow rules.

### Step 4 — Verify the fix:
After updating robots.txt, check that these 3 URLs are
not matched by any Disallow pattern.

- /blog?q&p=1, /blog?q&p=2, /blog?q&p=4, /blog?q&p=5
- /blog?q&p=6, /blog?q&p=7, /blog?q&p=8
- /blog?q&p=16, /blog?q&p=17, /blog?q&p=18

These waste crawl budget. Add noindex to all paginated pages.

### Find the blog listing component:
Look for: pages/blog.tsx, components/Blog.tsx, or similar

### Add noindex meta for paginated views:
```javascript
// In the blog listing page
const searchParams = new URLSearchParams(window.location.search)
const currentPage = parseInt(searchParams.get('p')) || 1

// In <head>:
{currentPage > 1 && (
  <meta name="robots" content="noindex, follow" />
)}
```

### Also add canonical for paginated pages:
```html
<link rel="canonical" href="https://purpleheartlimo.com/blog">
```
This should be added to ALL paginated views (p > 1).
