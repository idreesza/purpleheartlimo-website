### Step 2 — Add Content Signals at the TOP of robots.txt
Insert these 4 lines at the very beginning of the file,
BEFORE any existing User-agent rules:

```
# AI Content Signals — declares how AI systems may use this content
Content-Signal: ai-train=no
Content-Signal: search=yes
Content-Signal: ai-input=no
```

- ai-train=no → AI companies cannot use our content to train models

## TASK 3 — ADD MARKDOWN NEGOTIATION VIA _HEADERS FILE

Purple Heart Limo is deployed on Netlify. Netlify uses a special
`_headers` file to set HTTP response headers for all pages.

Markdown Negotiation means: when an AI agent requests a page with
the header `Accept: text/markdown`, the server responds that it
supports markdown format. This is a signal that tells AI crawlers
the site is "AI-agent friendly."

### Step 1 — Find or create the _headers file
Look for a file named `_headers` in the project root or /public.

If it already exists: ADD to it (do not overwrite existing rules).
If it doesn't exist: CREATE it.

### Step 2 — Add these headers to the _headers file:

```
# Global headers for all pages
/*
  X-Content-Type-Options: nosniff
  X-Frame-Options: DENY
  Link: </.well-known/api-catalog>; rel="api-catalog"

# Specific headers for AI agent content negotiation
/
  Vary: Accept
  Content-Type: text/html; charset=utf-8

# llms.txt specific headers
/llms.txt
  Content-Type: text/plain; charset=utf-8
  Cache-Control: public, max-age=86400

# robots.txt headers
/robots.txt
  Content-Type: text/plain; charset=utf-8
  Cache-Control: public, max-age=86400
```

### Step 3 — ALSO create a Markdown version of the homepage
Create a new file at the same level as index.html called
`index.md` OR serve the llms.txt content when `Accept: text/markdown`
is requested. For a static Netlify site, the simplest approach is
to ensure llms.txt exists and is linked from the homepage
(already done in Task 2).
